# STATUS LEGEND

Статус: базовая легенда для безопасной уборки репозитория.

Цель: чтобы человек, ChatGPT и Codex одинаково понимали, какие файлы активные, какие архивные, а какие требуют ручной проверки.

---

## Статусы

| Status | Meaning | Можно трогать автоматически |
|---|---|---|
| `active_runtime` | Живой рабочий результат конкретного проекта / товара | Нет, только после проверки задачи |
| `active_skill_library` | Переиспользуемый skill / инструкция / модуль | Нет, только точечные правки |
| `active_engine_rule` | Правило маршрутизации, роль, метод, шаблон системы | Нет, это влияет на весь pipeline |
| `archive_reference` | Архивный материал, который может быть полезен как источник | Нет, не удалять первым проходом |
| `prototype` | Экспериментальный скрипт, черновик, тестовая сборка | Только после ручной маркировки |
| `needs_review` | Непонятный файл или папка | Нет |
| `deprecated` | Устаревший файл, подтверждённый к выводу из активного маршрута | Можно переносить в архив, но не удалять без отдельного решения |

---

## Главное правило

```text
Если статус непонятен — ставим needs_review, а не archive и не deprecated.
```

---

## Запрет первого прохода

Первый проход уборки не удаляет файлы.

Разрешены только:

```text
создание карт
создание manifest-файлов
маркировка статусов
список proposed_move
```
