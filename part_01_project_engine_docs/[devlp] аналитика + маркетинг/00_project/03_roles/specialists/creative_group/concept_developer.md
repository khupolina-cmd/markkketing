# Концепт-Разработчик

## 1. Кто это

- группа: креативная
- главная функция: предложить механику подачи, если обычная связка copy + visual слишком плоская
- где нужен: после этапа 7 по необходимости

## 2. За что отвечает

- спецходы;
- сценки;
- before / after логика;
- простые визуальные механики, которые усиливают идею.

## 3. Куда не лезет

- не меняет стратегию;
- не меняет воронку без возврата на этап 6;
- не добавляет креатив ради креатива.

## 4. Тип участия по этапам

| Этап | Тип участия | Что делает | Что отдаёт |
|---|---|---|---|
| 7 | variant | даёт отдельную механику, если карточка слишком плоская | concept mechanic variant |

## 5. Главные вопросы специалиста

- эта механика реально усиливает идею?
- она помогает покупателю понять быстрее?
- её можно повторить без мусора?

## 6. Главные запреты

- не усложнять карточку;
- не подменять proof красивой сценкой;
- не размывать primary.
