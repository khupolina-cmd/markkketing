# Картошка: Подробный Этап 3A

Цель: показать не красивую итоговую матрицу, а именно технический черновик до специалистов.

Важно:

```text
это ещё не гипотезы
это ещё не выводы
это ещё не стратегия
```

Это слой, где мы честно показываем:

- из каких кусочков вообще собрали кандидатов;
- какие строки дошли до специалистов;
- какие вылетели;
- почему именно вылетели.

Главная рамка:

```text
мы не ищем объективную правду о картошке
мы собираем то, что покупатель может увидеть в отзывах как реальную причину купить или уйти
```

## 1. Что Входит В 3A

Для героя:

```text
Картошка "Домашняя", 2 кг
```

Источники:

- `02_signal_markup/hero/hero_potato_home/02__hero_potato_home__signal_markup.md`
- `01_intake/product_passports/hero/hero_potato_home/01__hero_potato_home__product_passport.md`
- `01_intake/01__potato_project__price_value_snapshot.md`

Для игрушечного примера считаем, что в паспорте товара уже зафиксированы простые факты:

```text
цена: 120 руб
вес: 2 кг
цена за кг: 60 руб
рейтинг: 4.6
отзывов: 800
обещание селлера: картошка для домашней кухни - суп, пюре, запекание
```

Важно:

```text
цена, вес, рейтинг, обещания тут не создают новые контракты сами по себе
они только модифицируют чтение того, что уже видно в отзывах
```

## 2. Из Чего Мы Вообще Собираем Кандидатов

Сначала не строим матрицу.

Сначала раскладываем отзывы на кусочки.

### Исходные Отзывы Героя

```text
H1. Картошка вкусная, быстро разваривается, беру для пюре.
H2. Попадается мелкая, но для супа нормально.
H3. Для жарки не подошла, разваливается.
H4. Грязная, долго мыть, больше не возьму.
H5. Недорогая, но попадаются зелёные клубни.
H6. Хорошо запекается, но надо выбирать крупнее.
```

### Сигнальные Кусочки

| Piece ID | Источник | Тип | Содержание |
|---|---|---|---|
| P1 | H1 | сценарий | беру для пюре |
| P2 | H1 | позитив / удержание | вкусная |
| P3 | H1 | поведение продукта | быстро разваривается |
| P4 | H2 | риск / ограничение | попадается мелкая |
| P5 | H2 | сценарий | для супа нормально |
| P6 | H3 | сценарий | для жарки |
| P7 | H3 | риск / ограничение | разваливается |
| P8 | H4 | риск | грязная |
| P9 | H4 | риск усилия | долго мыть |
| P10 | H4 | сильный негативный сигнал | больше не возьму |
| P11 | H5 | factual modifier | недорогая |
| P12 | H5 | риск качества | зелёные клубни |
| P13 | H6 | сценарий | хорошо запекается |
| P14 | H6 | ограничение | надо выбирать крупнее |

Что важно:

```text
один отзыв может дать несколько кусочков
один кусочек ещё не равен строке матрицы
строка матрицы появляется только когда мы собираем совместимую комбинацию
```

## 3. Как Мы Комбинируем Кусочки

Мы не делаем хаос вида:

```text
каждый кусок со всеми подряд
```

Иначе матрица правда раздуется в монстра.

Технические правила здесь такие:

### 3.1. Что разрешено

- сначала собирать кандидата внутри одного отзыва;
- потом осторожно усиливать кандидата кусочком из другого отзыва, если нет противоречия;
- добавлять цену, рейтинг, обещание селлера только как модификаторы чтения;
- оставлять спорные, но реалистичные строки для специалиста.

### 3.2. Что запрещено

- склеивать противоречащие друг другу куски;
- выдумывать новый мотив, которого нигде нет;
- превращать цену в волшебную индульгенцию;
- заранее решать за специалиста, важна строка или нет для стратегии.

### 3.3. Формула Одного Кандидата

Один кандидат в 3A - это:

```text
сценарий или контекст
+ видимый риск или ограничение
+ что рядом всё ещё удерживает покупку
+ factual modifiers
```

Если в строке нет хотя бы риска и опоры на evidence,

```text
она не живёт
```

## 5. Что Именно Было Отсеяно

Вот тут как раз тот кусок, которого нам не хватало.

### Вылетело как `drop_impossible`

Потому что строка противоречит уже имеющимся фактам:

- `C9` универсальная картошка для всего;
- `C10` чистая и без подготовки;
- `C11` крупный ровный калибр без отбора.

### Вылетело как `drop_duplicate`

Потому что это та же строка, только переименованная:

- `C12` дублирует `C2`.

### Не вылетело, а ушло в `fear_material / hold_for_specialist`

Потому что строка не запрещена законами мира, но у нас нет на неё опоры как на прямое обещание:

- `C13` "цена полностью оправдывает зелёные клубни".

Это очень важный тип.

```text
не невозможное
а просто не готовое к прямому обещанию
```

Такой материал не надо выбрасывать.

Его надо передать дальше как страх, риск ожидания или повод заранее обезвредить тревогу покупателя.

## 6. Что Уходит Специалистам

В рабочий слой `3B` уходят только:

- `keep`
- `hold_for_specialist`

То есть по картошке дальше живут:

- `C1`
- `C2`
- `C3`
- `C4`
- `C5`
- `C6`
- `C7`
- `C8`

Именно из них уже потом специалисты делают нормальную рабочую матрицу:

```text
кому это стоп
когда это терпимо
что удерживает
что не надо обещать
что усиливают конкуренты
```

## 7. Почему Здесь Пока Нет Гипотез

Потому что это не место для гипотез.

Тут ещё рано писать:

```text
люди воспримут это как деревенскую натуральность
люди могут простить это из-за ностальгии
люди сочтут это честной дешёвой картошкой
```

Это уже интерпретация.

Её делают позже специалисты.

Технический слой делает только три вещи:

1. собирает;
2. чистит;
3. не даёт потерять реалистичную строку слишком рано.

---

## 8. Что Важно Понять На Пальцах

### Ошибка старого подхода

Слишком рано показывать уже красивую матрицу.

Тогда непонятно:

- откуда взялись строки;
- что было отброшено;
- где был дубль;
- где была невозможная склейка;
- где строка спорная, но живая.

### Что даёт новый 3A

Он делает логику прозрачной:

```text
сигналы -> кусочки -> кандидаты -> технический отсев -> специалисты
```

Именно этого слоя нам и не хватало.

---

## 9. Мини-Вывод По Картошке

На картошке мы теперь видим:

- не 4 и не 5 строк из воздуха;
- а нормальный грязный набор кандидатов;
- часть из них вылетает как мусор;
- часть сохраняется;
- часть специально держится до человека.

То есть матрица больше не выглядит как магия.

Она выглядит как:

```text
честная сортировка материала перед специалистами
```
