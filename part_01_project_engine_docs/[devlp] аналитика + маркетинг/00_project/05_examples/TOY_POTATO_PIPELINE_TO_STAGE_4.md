# Игрушечный Прогон: Картошка До Стратегии

Цель: на очень простом примере понять, как идёт пайплайн от сырья до готовой мастер-матрицы контрактов героя.

Важно:

```text
по ролям и маршрутизации главным источником теперь считаем:
02_routing/READY_PIPELINE_01_05_RU.md
03_roles/ACTIVE_TEAM_GROUPS_AND_ROUTING_RU.md
```

Главный фокус:

```text
мы не ищем объективную правду о картошке
мы анализируем, что покупатель увидит в отзывах и где он отвалится
```

---

## Исходная Ситуация

Герой:

```text
Картошка "Домашняя", 2 кг
```

Прямые конкуренты:

```text
Картошка "Для жарки"
Картошка "Суповая"
```

Косвенные конкуренты:

```text
Пюре быстрого приготовления
Замороженная овощная смесь для супа
```

Почему пример удобный:

- герой и прямые конкуренты - одна категория;
- косвенные конкуренты показывают соседнюю логику выбора: быстрее, проще, без чистки.

# Этап 1. Intake / Сбор Сырья

## Что делаем

Просто раскладываем материалы:

- кто герой;
- кто прямой конкурент;
- кто косвенный конкурент;
- какие отзывы у нас есть;
- каких фактов не хватает.

## Сырьё Для Примера

### Герой: Картошка "Домашняя"

```text
H1. Картошка вкусная, быстро разваривается, беру для пюре.
H2. Попадается мелкая, но для супа нормально.
H3. Для жарки не подошла, разваливается.
H4. Грязная, долго мыть, больше не возьму.
H5. Недорогая, но попадаются зелёные клубни.
H6. Хорошо запекается, но надо выбирать крупнее.
```

### Прямой Конкурент: Картошка "Для жарки"

```text
D1-1. Крупная, ровная, для жарки супер.
D1-2. Дороговато, зато не разваливается.
D1-3. Внутри пару клубней гнилые, неприятно.
D1-4. Мыть почти не надо, чистая.
```

### Прямой Конкурент: Картошка "Суповая"

```text
D2-1. Мелкая, зато в суп удобно.
D2-2. Быстро варится, для пюре тоже норм.
D2-3. Для жарки не берите, разваливается.
D2-4. Дешёвая, но много глазков.
```

### Косвенный Конкурент: Пюре Быстрого Приготовления

```text
I1-1. Взяла, когда нет времени чистить картошку.
I1-2. Вкус не как домашнее, но удобно.
I1-3. Спасает на работе и в дороге.
I1-4. Слишком искусственный вкус, больше не беру.
```

### Косвенный Конкурент: Замороженная Смесь Для Супа

```text
I2-1. Для супа быстро, ничего чистить не надо.
I2-2. Картошка мелкая, но в супе нормально.
I2-3. Дороже обычных овощей, зато экономит время.
I2-4. Иногда попадается лёд, неудобно.
```

## Документ На Выходе

```text
01_intake/01__potato_project__intake_map.md
```

## Что важно

На этом этапе мы ещё ничего не решаем про стратегию.

Но уже фиксируем, если нам не хватает:

- цены;
- веса / объёма;
- цены за кг;
- рейтинга;
- количества отзывов;
- описания и обещаний селлера.

Без этого потом криво читается терпимость к минусу.

# Этап 3. Матрицы Контрактов

Это один человеческий этап.

Внутри него есть подшаги:

```text
3A технический черновик героя
3B рабочая матрица героя
3A технические черновики конкурентов
3B рабочие матрицы конкурентов
мастер-матрица героя
```

Подробный разбор именно грязного технического слоя смотри отдельно:

```text
05_examples/TOY_POTATO_03A_TECHNICAL_DRAFT.md
```

## Кто работает

### Техническая сборка по правилам

Делает:

- убирает невозможное;
- убирает дубли;
- связывает строки с evidence;
- добавляет цену / контекст;
- ставит флаг риска обещания.

Специалисты пока не работают.

Они получают матрицу на этапе 4.

### Важная поправка

Отдельного специалиста по честности здесь больше нет.

Эта функция встроена в:

- технический флаг `риск_обещания` на этапе матрицы;
- `owner + Маркетплейс-стратега` на этапах выводов и стратегии.

## Минимальная Форма Строки

| Контракт / сценарий | Видимый минус / риск | Кому стоп | Когда терпимо | Что удерживает покупку | Цена / контекст | Риск обещания | Evidence IDs | Статус |
|---|---|---|---|---|---|---|---|---|

Статусы:

```text
draft_from_hero
confirmed_by_direct
enriched_by_indirect
weakened_by_market
reject
hold
```

## Какие вопросы решает готовый этап 3

```text
1. Кто не наш:
   покупатель для жарки
   покупатель "без возни"

2. Где герой ещё жив:
   пюре
   частично суп

3. Что является терпимым минусом:
   мелкость для супа
   развариваемость для пюре

4. Что остаётся опасным:
   грязь / долго мыть
   зелёные клубни

5. Что нельзя обещать:
   универсальность
   чистоту
   крупный ровный калибр
   стабильное качество без оговорок
```

# Этап 4. Выводы По Мастер-Матрице

Теперь по мастер-матрице делаем один короткий файл:

```text
04_matrix_conclusions/hero/hero_potato_home/04__hero_potato_home__matrix_conclusions.md
```

Внутри две секции.

## 4.1. Фактические Выводы

Например:

```text
подтверждённые контракты:
- пюре - живая зона
- суп - возможно живая зона

не наш покупатель:
- жарка
- люди, которые хотят "без возни"

опасные минусы:
- грязь / долго мыть
- зелёные клубни

рискованные обещания:
- универсальная картошка для всех блюд
- чистая и лёгкая без подготовки
```

## 4.2. Интерпретационные Гипотезы

Например:

```text
гипотеза 1:
  interpretation: для части людей "грязная" может читаться не как ужас, а как обычная деревенская картошка
  confidence_level: heuristic
  why_it_matters: это может немного смягчить восприятие, если цена низкая

гипотеза 2:
  interpretation: на фоне косвенных альтернатив "без чистки" минус грязи будет читаться тяжелее
  confidence_level: evidence_based_inference
  why_it_matters: это повышает риск для покупателей, которые ценят скорость

гипотеза 3:
  interpretation: "быстро разваривается" может читаться как плюс, если человек уже мысленно варит пюре
  confidence_level: evidence_based_inference
  why_it_matters: это даёт живую линию для будущего позиционирования
```

## Кто тут работает

На этом этапе не делаем коллективный компромисс.

### Owner: Стратег позиционирования

Собирает итоговый файл выводов.

### Input: Поведенческий стратег

Даёт гипотезы чтения.

### Filter: Маркетплейс-стратег

Режет нерелевантное категории.

### Граница обещаний

Опасные интерпретации здесь режет не отдельная роль,
а связка:

- `owner`
- `Маркетплейс-стратег`

### Variant по необходимости: Креативный директор / креативный стратег

Не соавтор, а отдельный сильный вариант, если он реально нужен.

# Этап 5. Разработка Стратегии На Картошке

Теперь берём:

- мастер-матрицу героя;
- фактические выводы;
- интерпретационные гипотезы;
- авторскую матрицу стратегий.

И смотрим не "что вообще можно придумать", а:

```text
какое направление можно честно выбрать внутри твоего стиля
```

## 5.1. Что проверяем через авторскую матрицу

Допустимые линии:

- Характер бренда
- Выделиться
- Ясность
- Доверие
- Премиализация

Недопустимая по умолчанию линия:

```text
перекричать конкурента
```

Не потому что так никто не делает.

А потому что это не твой способ работы.

## 5.2. Что картошка реально может нести

По данным матрицы:

```text
пюре - да
суп - возможно
жарка - нет
"без возни" - нет
премиальный идеальный продукт - нет
```

## 5.3. Проверка Направлений

| Линия | Можно / нельзя | Почему |
|---|---|---|
| Характер бренда | ограниченно | можно только если у бренда реально есть выраженный характер; по одним отзывам картошки этого мало |
| Выделиться | осторожно | можно как support, но не через крик и не через враньё |
| Ясность | да | продукт читается через понятные сценарии: пюре / суп, а не "всё для всех" |
| Доверие | да | честная подача очень важна, потому что есть риски грязи и качества |
| Премиализация | нет | противоречит ценовой и продуктовой логике героя |

## 5.4. Выбор

### Основное направление

```text
Ясность
```

Почему:

- матрица уже показала, что товар нельзя продавать как универсальный;
- значит нужна ясная и простая рамка, где продукт действительно жив;
- самый честный путь — не расширять, а уточнять.

### Поддерживающее направление

```text
Доверие
```

Почему:

- у героя есть тревожные риски: грязь, зелёные клубни, возня;
- значит подача должна быть не вылизанная и не слишком рекламная;
- нужно не "докричать", а выглядеть живо и правдоподобно.

## 5.5. Чем Жертвуем

```text
мы жертвуем ощущением широты
мы не сможем продать картошку как "идеальную для всего"
мы не берём премиализацию
мы не идём в агрессивное отличие ради самого отличия
```

Но взамен получаем:

```text
честную и устойчивую стратегию
```

## 5.6. Что уходит дальше

```text
primary:
  ясность

support:
  доверие

do_not_do:
  не обещать универсальность
  не обещать чистоту и лёгкость без подготовки
  не строить премиальный образ
  не пытаться перекричать конкурентов
```

## Кто здесь решает

### Owner: Стратег позиционирования

Выбирает `primary + support`.

### Filter: Маркетплейс-стратег

Отрезает то, что не выдерживает категорию.

### Input: Поведенческий стратег

Даёт проверку человеческой правдоподобности.

### Граница обещаний

То, что требует опасных claims, здесь режет `Маркетплейс-стратег`.

### Variant по необходимости: Креативный директор

Если нужен альтернативный сильный ход, даёт `variant B`, а не усреднённый компромисс.
