# Промпт 01: Взвешивание сигналов (weight-signals)

## Назначение

Для каждой уникальной темы из `final_signal_registry` проставить три параметра:
- **Базовую severity** (только для негативных)
- **Reversibility** (только для негативных)
- **Evidence_strength** (для всех)

---

## Входные данные

```
final_signal_registry: [из skill 1]
stats: [из skill 1 — общее число отзывов, доли]
producer_claims: [список обещаний производителя из упаковки/брифа/карточки]
```

---

## Инструкция

### Шаг 1: Получить список уникальных тем

Из `final_signal_registry` извлечь уникальные темы (не отдельные отзывы — а темы, которые уже кластеризованы skill 1).

Для каждой темы известны:
- Категория (негативная / позитивная / сценарий / ожидание / сомнение)
- Число упоминаний (N_total)
- Число «чистых» упоминаний после фильтрации (N_clean)

### Шаг 2: Для негативных тем — проставить severity

Алгоритм:
1. Является ли тема угрозой здоровью / безопасности / правовым риском? → **critical**
2. Атакует ли тема **основную заявленную функцию** товара (проверить по `producer_claims`)? → **high**
3. Заметная проблема, товар частично работает? → **medium**
4. Мелкое неудобство, товар работает? → **low**

> Подсказка: Уровня «noise» не существует. Любой сигнал — хотя бы low.

Как проверять high:
- Взять тему негатива
- Найти в `producer_claims` обещания по этой же оси
- Если тема напрямую опровергает **главное** обещание → high
- Если опровергает **второстепенное** → medium

### Шаг 3: Для негативных тем — проставить reversibility

- `irreversible` — последствие необратимо (ожог кожи, постоянная аллергическая реакция)
- `fixable_with_cost` — устранимо, но требует усилий/денег (поход к врачу, возврат товара)
- `easily_fixable` — легко устранимо (промыть, нанести снова, использовать меньше)
- `self_resolving` — само проходит со временем

### Шаг 4: Для всех тем — проставить evidence_strength

- `strong` — конкретный опыт, детальное описание, специфические детали
- `moderate` — общее подтверждение без подробностей
- `weak` — упоминание без описания опыта («хороший», «понравился»)
- `suspect` — подозрительный отзыв: шаблонные фразы, нехарактерные для реального пользователя, возможная накрутка

### Шаг 5: Сформировать weighted_signal_map

---

## Формат выхода

```
=== WEIGHTED_SIGNAL_MAP ===

Тема | Кат. | N_total | N_clean | Severity | Reversibility | Evidence
-----|------|---------|---------|----------|---------------|--------
[тема 1] | neg | [N] | [N] | [critical/high/medium/low] | [тип] | [strength]
[тема 2] | pos | [N] | [N] | n/a | n/a | [strength]
[тема 3] | neg | [N] | [N] | [уровень] | [тип] | [strength]
...
```

---

## Правила и проверки

- [ ] Каждой теме проставлено значение evidence_strength
- [ ] Негативные темы имеют severity и reversibility
- [ ] Для позитивных тем severity = n/a, reversibility = n/a
- [ ] high проставлено только если тема атакует основную функцию (проверено по producer_claims)
- [ ] critical проставлено если есть угроза здоровью/безопасности
- [ ] Ни одной теме не присвоено severity = «noise» или пустое значение

---

## Анти-галлюцинаторные правила

1. Не добавлять темы, которых нет в `final_signal_registry`
2. Не менять категорию сигнала (позитив / негатив) — это уже определил skill 1
3. Не решать high/medium «на глаз» — всегда сверять с producer_claims
4. Если producer_claims не предоставлены — нельзя проставить high (severity по умолчанию = medium)
5. Не округлять N_total и N_clean
