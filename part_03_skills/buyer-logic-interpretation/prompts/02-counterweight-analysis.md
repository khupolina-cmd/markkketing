# Промпт 02: Противовесный анализ (counterweight-analysis)

## Назначение

На основе `weighted_signal_map` определить, по каким осям критериев товар можно ставить в ядро подачи, а по каким нельзя.

---

## Входные данные

```
weighted_signal_map: [из промпта 01]
```

---

## Инструкция

### Шаг 1: Определить оси критериев

Сгруппировать темы из `weighted_signal_map` по смысловым осям.

Типичные оси (не ограничиваться ими — брать из реальных данных):
- Запах / аромат
- Увлажнение / питание
- Текстура / консистенция
- Стойкость эффекта
- Состав / безопасность
- Упаковка / дозатор / внешний вид
- Цена / объём / расход
- Результат (внешний вид, эффект)
- Доставка / сервис (если есть в данных)

Одна тема может входить только в **одну** основную ось.

### Шаг 2: Для каждой оси — собрать позитив и негатив

Разделить темы по оси на:
- Позитивные (cat = pos) — список с N_clean и evidence
- Негативные (cat = neg) — список с N_clean, severity и evidence

### Шаг 3: Рассчитать частотный баланс

```
позитивных_упоминаний = сумма N_clean всех позитивных тем по оси
негативных_упоминаний = сумма N_clean всех негативных тем по оси
balance = позитивных / негативных (если негативных > 0)
```

Если негативных = 0 → balance = ∞ (безусловный позитив).

### Шаг 4: Определить max severity негативов

Взять максимальный уровень severity среди всех негативных тем по оси.

### Шаг 5: Проверить правило critical

Если хотя бы одна негативная тема имеет severity = **critical** → ось автоматически блокирована (🔴), независимо от частотного баланса.

### Шаг 6: Вынести вердикт

| Условие | Вердикт |
|---|---|
| Менее 5 сигналов по оси суммарно | ⚫ Нет данных |
| Есть хоть один critical | 🔴 Нельзя в ядро |
| Balance < 2:1 | 🔴 Нельзя в ядро |
| Balance 2:1–3:1 ИЛИ max severity = high | 🟡 Спорное |
| Balance 3:1–4:1, max severity ≤ medium | ⚠️ Условно рабочее |
| Balance ≥ 4:1, max severity ≤ medium, evidence позитива strong | ✅ Сильная опора |

### Шаг 7: Дать рекомендацию

Для каждой оси — краткая рекомендация о том, как работать с этой осью в карточке.

## Правила и проверки

- [ ] Все темы из weighted_signal_map распределены по осям
- [ ] Критическое правило применено: один critical → 🔴 для оси
- [ ] Частотный баланс посчитан на основе N_clean, не N_total
- [ ] Вердикт соответствует таблице решений
- [ ] Для каждой оси есть рекомендация

---

## Анти-галлюцинаторные правила

1. Не добавлять темы к осям, которых нет в weighted_signal_map
2. Не повышать вердикт оси, если есть critical (critical всегда блокирует)
3. Не объединять оси произвольно — группировка должна быть смысловой
4. Если balance = ∞ (нет негатива) — честно указать «негативных сигналов по оси не зафиксировано», но не присваивать ✅ автоматически, если данных мало (⚫)
