# Промпт 03: Карта безопасности обещаний (promise-safety)

## Назначение

Для каждого обещания производителя определить, насколько безопасно его использовать в карточке товара — с учётом реального цифрового следа.

---

## Входные данные

```
weighted_signal_map: [из промпта 01]
producer_claims: [список обещаний производителя]
```

---

## Инструкция

### Шаг 1: Перечислить все обещания производителя

Взять полный список `producer_claims`.

Если обещаний нет → зафиксировать: «producer_claims не предоставлены — анализ невозможен».

### Шаг 2: Для каждого обещания — определить ось

Какой оси критерия соответствует это обещание?  
(Увлажнение, запах, состав, стойкость, текстура и т.д.)

### Шаг 3: Собрать три группы сигналов

Из `weighted_signal_map` найти сигналы, связанные с той же осью:

**Подтверждения (confirmation):**
- Позитивные темы по этой оси
- Для каждой: N_clean и evidence_strength

**Опровержения (contradiction):**
- Негативные темы по этой оси
- Для каждой: N_clean, severity и evidence_strength

**Ожидания (expectation):**
- Темы из категории «ожидания» по этой оси (из final_signal_registry)
- Для каждой: N_clean

### Шаг 4: Вынести вердикт

| Условие | Вердикт |
|---|---|
| Есть хоть один critical в опровержениях | ⛔ Запрещено |
| Сильные опровержения (severity = high), значимая доля | 🔴 Опасно |
| Есть опровержения (severity ≤ medium), значимая доля | ⚠️ Упоминать, не в центр |
| Опровержения есть, но слабые; часть не получает | 🟡 Можно с оговоркой |
| Подтверждения сильные, опровержения минимальные | ✅ Безопасно |
| Данных по оси менее 5 сигналов | ⚫ Нет данных |

### Шаг 5: Сформулировать обоснование

Для каждого обещания — 1–2 предложения, объясняющие вердикт.

---

## Формат выхода

```
=== PROMISE_SAFETY_MAP ===

=== Обещание: «[текст]» (из [источника]) ===

ПОДТВЕРЖДЕНИЯ:
- «[тема]»: N упоминаний, evidence = [strength]
[Или: Подтверждений не найдено]

ОПРОВЕРЖЕНИЯ:
- «[тема]»: N упоминаний, severity = [уровень], evidence = [strength]
[Или: Опровержений не найдено]

ОЖИДАНИЯ:
- «[тема]»: N упоминаний
[Или: Ожиданий по этой оси не зафиксировано]

ВЕРДИКТ: [✅/🟡/⚠️/🔴/⛔/⚫]
ОБОСНОВАНИЕ: [текст]

---

[Повторить для каждого обещания]

=== СВОДКА ===
✅ Безопасно: [список]
🟡 С оговоркой: [список]
⚠️ Не в центр: [список]
🔴 Опасно: [список]
⛔ Запрещено: [список]
⚫ Нет данных: [список]
```

---

## Правила и проверки

- [ ] Каждое обещание из producer_claims проверено
- [ ] Для каждого обещания найдены (или честно указано «нет») подтверждения и опровержения
- [ ] critical в опровержениях → автоматически ⛔
- [ ] Обоснование написано для каждого обещания
- [ ] Сводка заполнена

---

## Анти-галлюцинаторные правила

1. Не приписывать сигналы к обещанию, если они по другой оси
2. Не игнорировать опровержения, даже если их мало
3. Не ставить ✅ при нулевых данных — это ⚫
4. Если обещание не связано ни с одной темой из данных → ⚫ (нет данных), не ✅
