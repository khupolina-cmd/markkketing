# Промпт 05: Извлечение профилей и диагностических линз

## Назначение

Собрать из `final_signal_registry`:

- наблюдаемые профили;
- диагностические линзы.

Это не сегментация ради разных воронок.

Это подготовка линз для стресс-теста общей подачи.

## Входные данные

```text
final_signal_registry: [из skill 1 — категории: сценарии, ожидания, сомнения]
stats: [из skill 1]
```

## Инструкция

### Шаг 1: Собрать observed profiles

Из сценариев, ожиданий и сомнений выделить реальные повторяющиеся паттерны:

`зачем + что хотели + чего боялись`

Сгруппировать их в `observed profiles`.

### Шаг 2: Собрать diagnostic lenses

Проверить, есть ли скрытые чувствительности, которые:

- не обязаны быть массовыми;
- но нужны для стресс-теста общей подачи.

Примеры:

- чувствительный к оригинальности;
- скептик к overclaim;
- чувствительный к цене ошибки;
- чувствительный к подарочному стыду.

### Шаг 3: Проверить полезность

Любая линза должна:

- менять интерпретацию хотя бы одного сигнала;
- вскрывать скрытый барьер или angle opportunity;
- быть обоснована цитатой или повторяющимся паттерном.

### Шаг 4: Ограничить количество

- минимум 2 линзы;
- максимум 6 линз.

Если данных мало — честный fallback.

## Формат выхода

```text
=== PERSONA_PROFILES ===
Всего линз: N

=== Профиль/линза: [название] ===
Класс: [observed / diagnostic]
Доля: ~[X]% от массива

Мотив:
...

Главный страх:
...

Вторичные ожидания:
...

Ключевые цитаты:
- «...»
- «...»

Как меняет интерпретацию:
...

Что добавляет в общую воронку:
...
```

## Анти-галлюцинаторные правила

1. Не придумывать линзы без опоры на данные
2. Не плодить линзы без пользы
3. Не путать линзы с готовыми сегментными стратегиями
4. Если данных мало — честный fallback
