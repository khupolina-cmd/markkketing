# Матрица безопасности обещаний (promise_safety_map)

## Назначение

Для каждого обещания производителя определить, насколько безопасно его использовать в карточке товара с учётом реального цифрового следа.

**Вопрос:** Если мы поставим это обещание в ядро карточки — создаст ли оно разочарование или риск?

---

## Три измерения проверки

Для каждого обещания из `producer_claims` нужно собрать три группы сигналов из `weighted_signal_map`:

### Подтверждение (confirmation)
Позитивные сигналы по той же оси. Подтверждают, что обещание выполняется.

Пример: обещание «глубокое увлажнение» → позитивные сигналы об увлажнении.

### Опровержение (contradiction)
Негативные сигналы по той же оси. Противоречат обещанию.

Пример: обещание «стойкий аромат» → негативные сигналы о том, что запах быстро исчезает.

### Ожидание (expectation)
Сигналы из категории «ожидания» (из skill 1), связанные с этой осью. Показывают, что покупатель хотел получить.

Пример: обещание «нежная текстура» → ожидания о том, что крем будет невесомым.

---

## Матрица решений

### ✅ Безопасно обещать
**Условия:**
- Сильные подтверждения (evidence = strong, высокая частота)
- Минимальные или нулевые опровержения (severity ≤ low)
- Ожидания совпадают с тем, что получают

**Действие:** Можно ставить в ядро подачи.

---

### 🟡 Можно с оговоркой
**Условия:**
- Подтверждения есть, но часть покупателей не получает обещанного
- Опровержения есть (severity ≤ medium)
- Ожидания могут быть завышены

**Действие:** Упоминать, но уточнять условия или добавить «для большинства кожи».

---

### ⚠️ Упоминать, не в центр
**Условия:**
- Обещание частично работает
- Значимая доля опровержений (severity = medium/high)
- Есть риск разочарования

**Действие:** Можно упомянуть, но не делать центральным аргументом. Не обещать категорично.

---

### 🔴 Опасно — создаёт разочарование
**Условия:**
- Опровержения сильные и частые
- Severity = high
- Обещание противоречит значимой доле опыта

**Действие:** Не ставить в ядро. Если упоминать — только с очень явными оговорками. Предупредить заказчика.

---

### ⛔ Запрещено
**Условия:**
- Есть хотя бы один сигнал с severity = critical, связанный с этим обещанием
- ИЛИ обещание категорически противоречит массовому опыту

**Действие:** Нельзя использовать это обещание в карточке без дисклеймера. Жёсткое предупреждение заказчику.

---

## Особые случаи

### Обещание «гипоаллергенный» / «для чувствительной кожи»
Если есть хотя бы один отзыв об аллергической реакции → автоматически ⛔ для этого конкретного обещания.

Логика: Обещание «гипоаллергенный» напрямую касается безопасности. Один сигнал о вреде опровергает его на юридическом и этическом уровне.

### Обещание без опровержений
Если опровержений нет совсем, но данных мало (< 5 сигналов по оси) → вердикт ⚫ (нет данных). Нельзя считать это подтверждением.

---

## Формат выхода: promise_safety_map

```
=== Обещание: «[текст обещания]» (из [источника: упаковка/бриф/карточка]) ===

ПОДТВЕРЖДЕНИЯ:
- [сигнал]: N упоминаний, evidence = [strength]
[или: Подтверждений не найдено]

ОПРОВЕРЖЕНИЯ:
- [сигнал]: N упоминаний, severity = [уровень], evidence = [strength]
[или: Опровержений не найдено]

ОЖИДАНИЯ:
- [ожидание из данных]: N упоминаний
[или: Ожиданий по этой оси не зафиксировано]

ВЕРДИКТ: [✅/🟡/⚠️/🔴/⛔]
ОБОСНОВАНИЕ: [краткое пояснение решения]
```

**Пример:**

```
=== Обещание: «глубокое увлажнение 24 часа» (из упаковки) ===

ПОДТВЕРЖДЕНИЯ:
- «кожа стала мягкой после первого применения»: 47 упоминаний, evidence = strong
- «увлажняет хорошо»: 31 упоминание, evidence = moderate

ОПРОВЕРЖЕНИЯ:
- «эффект держится 3–4 часа, не 24»: 18 упоминаний, severity = high, evidence = strong
- «не увлажняет совсем, кожа сухая»: 9 упоминаний, severity = high, evidence = strong

ОЖИДАНИЯ:
- «хочу, чтобы кожа была увлажнённой весь день»: 22 упоминания

ВЕРДИКТ: ⚠️ Упоминать, не в центр
ОБОСНОВАНИЕ: Значительная доля (около 23%) не получает обещанных 24 часов. Ожидание «весь день» широко распространено, но не оправдывается. Риск разочарования высокий.
```
