# Skill 2: Кластеризация по толерантности к минусам — инструкция для запуска

## Что это

Инструмент, который берёт таблицу сигналов из отзывов (результат Skill 1) и отвечает на главный вопрос: **«Для кого наш продукт идеален — включая его недостатки?»**

Skill 2 находит группы покупателей, для которых минусы продукта — не минусы. Эти группы называются **кластерами**. Кластер — это не «тип человека», а **пучок сигналов**, которые встречаются вместе в нескольких отзывах.

**Что Skill 2 не делает:** не даёт рекомендаций по карточке, не пишет офферы, не советует что менять. Это чистая аналитика — материал для Инструмента 3.

---

## Что подготовить

### Обязательно

**`signal-markup.md`** — результат Skill 1.  
Таблица отзывов с размеченными сигналами (🟢 позитивные, 🔴 негативные, 🟡 нейтральные). Если ещё не делал — запусти Skill 1 сначала.

### По желанию

**`card-review.md`** — краткий обзор карточки товара.  
Нужен как контекст: помогает понять отзывы, которые реагируют на конкретные обещания карточки. Создай его самостоятельно: запиши, что написано в карточке на маркетплейсе — обещания, образы, буллиты, описание.

**`prisms-library.md`** — кластеры от предыдущего товара.  
Нужен, если уже запускал Skill 2 раньше и хочешь проверить, работают ли те же кластеры для нового товара. После каждого прогона Skill 2 создаёт этот файл автоматически (Секция 7 вывода).

## Как запустить

### Шаг 1: Открой LLM

Подойдёт ChatGPT (GPT-4 или выше), Claude (Sonnet или Opus) или другая мощная языковая модель.

### Шаг 2: Скопируй промпт

Открой файл `skill-2-perception-prisms.prompt.md` и скопируй **весь** его текст.

### Шаг 3: Вставь промпт

Вставь скопированный текст в чат как первое сообщение (или как системный промпт, если интерфейс поддерживает).

### Шаг 4: Приложи файлы

Прикрепи к чату:
- `signal-markup.md` (обязательно)
- `card-review.md` (если есть)
- `prisms-library.md` (если есть — активирует режим «с багажом»)

### Шаг 5: Запусти

Напиши в чат:

```
Запусти Skill 2
```

### Шаг 6: Дождись результата

Прогон займёт несколько минут. Если модель остановилась — напиши «продолжай».

### Шаг 7: Сохрани результат

Скопируй весь результат в файл `clusters-[дата].md`.

Отдельно скопируй **Секцию 7 («Файл кластеров для переноса»)** → сохрани как `prisms-library.md` для следующего прогона.

## Как читать результат

**Сводная таблица кластеров** — начни отсюда. Показывает все найденные кластеры, их размер и ключевые сигналы.

**Карточки кластеров** — подробно по каждому кластеру:
- **●** = сильный кластер, 5+ отзывов → основа для карточки
- **◐** = слабый кластер, 3–4 отзыва → учитывать, но с осторожностью
- **Терпимые минусы** — что эти люди терпят и не считают проблемой
- **Стоп-факторы (гипотеза)** — что вероятно оттолкнуло бы
- **Серая зона** — молчаливые покупатели с таким же пучком

**«Не наша» аудитория** — кого не нужно привлекать карточкой.

**Conflict Map** — где разные кластеры хотят противоположного. Это самое важное: здесь карточка должна делать выбор.

**Калибровка ключевых сигналов** — насколько доверять восторженным отзывам.

**Ограничения данных** — что нельзя утверждать на основе этих данных.

## Как прогонять конкурентов

Полная пошаговая инструкция — в отдельном файле: **[competitor-workflow.md](competitor-workflow.md)**

### Три режима в зависимости от типа конкурента

**Прямые конкуренты → ВАЛИДАЦИЯ**  
Запусти Skill 1 по отзывам конкурента → получи `[конкурент]-signal-markup.md` → запусти Skill 2 **без** `prisms-library.md` (режим «с нуля», чтобы избежать confirmation bias). Потом вручную сравни кластеры конкурента с нашими. Подтверждённые кластеры обновляются в `prisms-library.md` (● → ●● → ●●●).

**Косвенные конкуренты → РАСШИРЕНИЕ**  
Не переносить кластеры конкурента напрямую. Вопрос: есть ли у них пучок сигналов, которого у нас нет? Новый кластер — только если в наших данных есть 3+ подтверждающих отзыва.

**Косвенные конкуренты → ИМПОРТ ЯЗЫКА**  
Формулировки и сценарии из отзывов косвенного конкурента — в файл `angles-library.md` для Инструмента 3. Не переносятся в карточки кластеров.

### Порядок прогонов

1. Первый прогон — свой товар: `signal-markup.md` → сохрани `prisms-library.md`
2. Прогоны прямых конкурентов: `[конкурент]-signal-markup.md` → сравни кластеры → обнови `prisms-library.md`
3. Прогоны косвенных конкурентов: расширение гипотез + `angles-library.md`
4. Финальный прогон товара-героя с багажом: `signal-markup.md` + `prisms-library.md` → финальные кластеры v2

Кластер считается «кластером категории» (●●●) если подтверждён у 3+ товаров.

## Как корректировать результат

Если кластер кажется неверным:
```
Кластер «[название]» кажется слишком широким. Уточни, что именно отличает этот пучок от «[другой кластер]».
```

Если хочешь проверить гипотезу:
```
Проверь: есть ли в данных кластер с пучком [сигнал 1 + сигнал 2]? Найди якорные отзывы или объясни, почему нет.
```

Если нужны цитаты по кластеру:
```
По кластеру «[название]» нужны полные цитаты якорных отзывов с ID.
```

Если хочешь уточнить Conflict Map:
```
По конфликту параметра «[параметр]» — есть ли отзывы, где человек упоминает оба желания одновременно?
```

## Что делать с результатом дальше

Результат Skill 2 — входные данные для **Инструмента 3 (стратегия слайдов)**:
- Карточки кластеров → определяют, на кого строится карточка
- Conflict Map → определяет, какой выбор делает карточка
- «Не наша» аудитория → определяет, кого карточка не должна привлекать

Офферы, углы подачи, запретные слова, кросс-категорийные идеи — всё это задача Инструмента 3, не Skill 2.

## Частые вопросы

**Сколько кластеров обычно получается?**  
2–6. Меньше двух — данных мало или все пишут про одно и то же. Больше шести — скорее всего, пучки пересекаются и их можно объединить.

**Что делать, если все кластеры с уверенностью ◐?**  
Данные не дают сильных пучков — либо отзывов мало, либо они однородны. Используй кластеры как гипотезы и отметь это в Инструменте 3.

**Нужно ли прогонять каждый раз заново?**  
При накоплении 50+ новых отзывов или перед крупным изменением карточки.

**Почему нет процентов?**  
Отзывы пишут ~5% покупателей с перекосом к полюсам. Процент от отзывов не равен проценту покупателей — это создаёт иллюзию точности там, где её нет.
