# Skill: perception-prisms

## Назначение

Skill `perception-prisms` берёт реестр сигналов из Skill 1 и отвечает на вопрос: **«Для кого наш продукт идеален — включая его недостатки?»**

Кластеризация идёт по **ко-встречаемости сигналов** в отзывах, а не по домысленным мотивациям. Кластер — это пучок факторов, а не тип человека.

Главный результат — карта кластеров и Conflict Map для Инструмента 3 (стратегия слайдов). Skill 2 не даёт рекомендаций по карточке.

## Когда использовать

- есть `signal-markup.md` — результат Skill 1;
- нужно понять, для кого минусы продукта — не минусы;
- нужна карта кластеров для построения стратегии карточки.

Не использовать этот skill для:

- первичной разметки сигналов (это Skill 1);
- написания офферов и рекомендаций по карточке (это Инструмент 3);
- построения разрывов «карточка ↔ реальность» (отдельный прогон).

## Главная архитектурная идея

Не «какие плюсы важны», а **«для кого наши минусы — не минусы»**. Это определяет, на кого строить карточку.

Кластер подтверждается только при ко-встречаемости 3+ сигналов в 3+ отзывах. Проценты не используются — только количество отзывов.

## Вход

**Обязательные:**
1. `signal-markup.md` — результат Skill 1

**Опциональные:**
2. `card-review.md` — контекст карточки товара (не модуль, только для понимания отзывов, реагирующих на карточку)
3. `prisms-library.md` — кластеры от предыдущего товара (режим «с багажом»)

Пример формата: [examples/prisms-library-example.md](examples/prisms-library-example.md)

## Выход

Skill возвращает 7 секций:

1. **Сводная таблица кластеров** — все кластеры, размер, драйверы, терпимые минусы, стоп-факторы
2. **Карточки кластеров** — подробно по каждому: пучок, терпимые минусы, стоп-факторы (гипотеза), явные мотивации, маркеры входа, серая зона, якорные отзывы
3. **«Не наша» аудитория** — кластеры, для которых объективные свойства продукта = стоп-фактор
4. **Conflict Map** — где кластеры конфликтуют по одному параметру продукта
5. **Калибровка ключевых сигналов** — насколько доверять восторженным отзывам
6. **Ограничения данных** — что нельзя утверждать
7. **Файл кластеров для переноса** (prisms-library.md)

## Pipeline flow

```text
Input: signal-markup.md [+ card-review.md (контекст)] [+ prisms-library.md]
    ↓
[ШАГ 0] → заземление: тон А/Б/В, калибровочные отзывы, маркеры входа
    ↓
[ШАГ 1] → выделение информативных отзывов (2+ сигнала), профили
    ↓
[ШАГ 2] → кластеризация по пучкам (ко-встречаемость 🟢 сигналов)
    ↓
[ШАГ 3] → «не наши» — кластеры со стоп-факторами = свойства продукта
    ↓
[ШАГ 4] → серая зона внутри каждого кластера
    ↓
[ШАГ 5] → Conflict Map
    ↓
[ШАГ 6] → калибровка через «тишину»
    ↓
[ФИНАЛ] → 7 секций вывода + prisms-library.md
```

## Два режима

**«С нуля»** (нет `prisms-library.md`):
- Кластеры строятся из данных текущего товара
- Уверенность: ● (5+ отзывов) / ◐ (3–4 отзыва)

**«С багажом»** (есть `prisms-library.md`):
- Прямые конкуренты → ВАЛИДАЦИЯ существующих кластеров
- Косвенные → РАСШИРЕНИЕ (новый кластер только при 3+ маркерах в наших данных)
- Заменители → ИМПОРТ ЯЗЫКА в Инструмент 3

Кластер переносится только если подтверждён в 3+ товарах.

## Ключевые принципы

1. **Кластер = пучок факторов, не тип человека.**
2. **Главное — толерантность к минусам.**
3. **Не домысливать мотивации** — только явные («купила в подарок»).
4. **Проценты — ложь.** Только количество отзывов.
5. **Тишина ≠ согласие.** Редкий негатив может быть недооценён.
6. **Серая зона — внутри каждого кластера**, не отдельная призма.
7. **Никаких офферов и рекомендаций** — это задача Инструмента 3.

## Жёсткие запреты

1. Не создавать демографические портреты («женщина 35 лет»).
2. Не использовать проценты (только количество отзывов).
3. Не домысливать мотивации без явного высказывания автора отзыва.
4. Не давать рекомендации по карточке, офферы, запретные слова.
5. Не придумывать сигналы — только из `signal-markup.md`.
6. Не строить кластер при менее 3 подтверждающих отзывах.
