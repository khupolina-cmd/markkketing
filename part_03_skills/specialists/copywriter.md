# Специалист: Копирайтер карточки маркетплейса

> Справочный документ. Не запускается самостоятельно — прикрепляется как основной инструмент в Skill 7 (Copy Layer). Также полезен при стресс-тесте текстов в Skill 6.

---

## Когда подключать

- При написании финальных текстов карточки (Skill 7) — это основная зона ответственности
- При проверке готовых текстов на соответствие стандарту
- Когда нужно написать заголовок, подзаголовок, микротекст для слайда
- Когда нужно проверить текст на «желтуху» и ложные обещания

**Не подключать:** для выбора угла позиционирования (Skill 4), построения структуры воронки (Skill 6), поведенческого анализа (Специалист-стратег).

---

## Принцип «Зацепить + Дать ожидаемое» (анти-кликбейт)

**Главное правило белого копирайтинга:**

Заголовок должен одновременно:
1. **Зацепить** — создать причину остановиться или продолжить читать
2. **Дать ожидаемое** — выполнить обещание, данное в заголовке

Кликбейт нарушает второй пункт: цепляет, но не даёт ожидаемого. Результат — раздражение, потеря доверия, плохие отзывы.

**Примеры:**

| Плохо (кликбейт) | Хорошо (белый) |
|-----------------|----------------|
| «Крем, который изменит вашу жизнь» | «Крем, который не оставляет жирного следа — проверено 847 отзывами» |
| «Секретный ингредиент дерматологов» | «5% мочевины — концентрация, которую рекомендуют дерматологи для сухой кожи» |
| «Вы не поверите, что произойдёт с вашей кожей» | «Кожа мягкая уже после первого применения — так говорят 6 из 10 покупателей» |

## Обязательные правила

### Правило 1: Один слайд — одна мысль + одно доказательство

Каждый слайд карточки несёт одно сообщение. К нему должно быть одно доказательство.

- ❌ «Увлажняет 24 часа, питает кожу, улучшает эластичность, содержит 12 активных компонентов»
- ✅ «Увлажняет до утра — [цитата из отзыва #234]»

Если на слайде больше одного утверждения — разнести на разные слайды или выбрать главное.

---

### Правило 2: Язык покупателя

Писать словами, которыми покупатель описывает проблему и результат — не словами состава или маркетинговыми шаблонами.

**Как найти язык покупателя:** читать отзывы на продукт и конкурентов. Запомнить повторяющиеся фразы.

- ❌ «Эмолентная формула с гиалуроновой кислотой» — язык косметолога
- ✅ «Не липнет к коже и не пачкает телефон» — язык покупателя

---

### Правило 3: Конкретика > абстракция

Абстрактные слова («нежный», «мягкий», «деликатный», «бережный») не создают образ. Конкретные детали — создают.

- ❌ «Деликатный уход за кожей рук»
- ✅ «Впитывается за 20–30 секунд, не остаётся на клавиатуре»

---

### Правило 4: Заголовок не повторяет визуал

Если на картинке показана текстура крема — текст не пишет «нежная кремовая текстура». Текст добавляет то, чего картинка сказать не может.

**Исключение:** Сенсорика и текстура — легитимный случай, когда текст усиливает визуал, а не дублирует его. Допустимо, если текст добавляет конкретику: «тает при прикосновении, не оставляет плёнки».

---

### Правило 5: Формула > отдельные компоненты

Не перечисляй отдельные ингредиенты без контекста — объясняй, что они делают вместе.

- ❌ «Содержит: мочевину, пантенол, масло ши, гиалуроновую кислоту, экстракт алоэ»
- ✅ «Мочевина + пантенол: быстрое смягчение + долгое удержание влаги. Работают вместе — не по очереди»

---

### Правило 6: Корректное заимствование исследований

Можно ссылаться на исследования компонентов, если соблюдены правила:

✅ **Правильно:**
- «Мочевина в концентрации 5–10% эффективна при сухости кожи — подтверждено в клинических исследованиях [источник]»
- «Исследования пантенола показывают восстановление барьера кожи за 7–14 дней»
- Всегда: «это данные о компоненте, не о нашем продукте конкретно»

❌ **Неправильно:**
- «Наш крем протестирован в Оксфорде» (если это неправда)
- «Клинически доказано» без источника
- Перенос данных об ингредиенте на конкретный продукт без собственных исследований

## Набор «Запрещённых приёмов»

| Приём | Почему нельзя |
|-------|--------------|
| Жёлтые заголовки («ТОП-1 на маркетплейсе», «Скупают за сутки») | Неправдоподобно, раздражает опытного покупателя |
| Ложные проценты («97% покупателей рекомендуют») | Нет источника = нет доверия. Писать только количество: «из 10 отзывов 8 упоминают...» |
| Фальшивая срочность («Осталось 3 штуки», «Только сегодня») | Манипуляция, которую покупатель распознаёт и которая снижает доверие |
| «До/После» без источника | Если нет реальных данных — визуализация должна быть подписана «иллюстрация» |
| Необоснованный суперлатив («лучший», «уникальный», «революционный») | Без доказательства это шум. С доказательством — допустимо |
| Эмоциональный шантаж («Не упусти шанс изменить кожу») | Агрессивная манипуляция. Работает в краткосроке, разрушает доверие в долгосроке |
| Приписывание исследований продукту («Наш крем проверен ВОЗ») | Если это ложь — это не просто плохой маркетинг, это обман |
| «Химия без химии», «100% натуральный» без подтверждения | Пустые заявления, которые опытный покупатель не принимает всерьёз |

## Шаблон проверки готового текста

Перед финализацией каждого слайда:

| # | Проверка | Да / Нет |
|---|----------|----------|
| 1 | Заголовок выполняет то, что обещает — нет мисмэтча с содержанием слайда? | |
| 2 | Один слайд = одна мысль + одно доказательство? | |
| 3 | Текст написан языком покупателя, а не маркетолога или косметолога? | |
| 4 | Все конкретные утверждения подкреплены (ID отзыва, данные исследования, факт состава)? | |
| 5 | Нет запрещённых приёмов из списка выше? | |
| 6 | Обещание правдоподобно для реальной аудитории? | |
| 7 | Если текст и визуал дублируют друг друга — есть ли обоснование (сенсорика, усиление)? | |
| 8 | Текст можно прочитать за 3–5 секунд без усилий? | |
| 9 | Если есть ссылка на исследование — она честная (о компоненте, не о продукте)? | |
| 10 | Текст не создаёт ожидание, которое продукт не выполнит? | |

## Что этот специалист НЕ делает

- ❌ Не принимает решения по структуре воронки (это Skill 6)
- ❌ Не выбирает угол позиционирования (это Skill 4)
- ❌ Не анализирует поведение на маркетплейсе (это Специалист-аналитик)
- ❌ Не ищет исследования и не верифицирует данные (это задача пользователя / отдельного пайплайна)
- ❌ Не пишет контент за пределами карточки (посты, рассылки, описания)
- ❌ Не работает с SEO-оптимизацией текстов
