Ввод:

<<<

{СИГНАЛЫ}

HTML-массив отзывов / вопросов:

<<<

{HTML_МАССИВ}

Ниже я даю тебе уже зафиксированные сигналы в формате:

имя | категория | код сигнала

Пример:

Анна | негатив | слаб.увл.

Мария | негатив | резк.зап.

Твоя задача — не искать сигналы заново, а работать только с уже размеченными сигналами, которые вставлены в блок {СИГНАЛЫ}.

#### Что нужно сделать

Возьми только сигналы категории негатив.

Для каждого сигнала:

найди соответствующий отзыв / вопрос в HTML-массиве по имени;

разверни полный отзыв / вопрос целиком;

внутри полного текста жирным выдели только тот кусок, который соответствует сигналу;

если у одного и того же имени несколько негативных сигналов, разворачивай их по отдельности, а не одним блоком;

если в одном и том же отзыве есть несколько негативных сигналов, каждый разворачивай отдельно, с тем же полным текстом, но с новым выделением.

После полного разворота:

сгруппируй сигналы в темы по 2–4 слова;

посчитай частоты тем и отсортируй по убыванию;

посчитай B = общее число всех отзывов / вопросов в HTML-массиве;

для каждого финального тега посчитай % = (N / B) × 100.

#### Правила подсчёта

B = всё количество отзывов / вопросов в HTML-массиве, а не количество найденных негативных сигналов;

N для темы = сколько разных отзывов / вопросов содержат этот конкретный негативный сигнал или тему;

если один и тот же сигнал повторяется в одном отзыве несколько раз, считай это как 1 отзыв;

если в одном отзыве есть 3 разных негативных сигнала, каждый идёт в свой сигнал / тему отдельно;

проценты всегда считать от B, а не от числа сигналов и не от числа негативных отзывов.

#### Правила работы

не переопределяй категорию сигнала;

не ищи новые сигналы;

не додумывай другие куски;

работай только с теми сигналами, которые вставлены в блок {СИГНАЛЫ};

если один и тот же отзыв разворачивается по нескольким сигналам, каждый раз показывай тот же полный текст, но с новым выделением;

не сокращай цитаты;

не пересказывай отзыв своими словами;

если имя в сигнале указано как «Покупатель», используй именно «Покупатель»;

если по имени в HTML найдено несколько отзывов и нельзя однозначно понять, какой относится к сигналу, не гадай и пометь как спорный случай в блоке ручной проверки.

#### Формат ответа

A. Все сырые негативные сигналы

Для каждого сигнала:

имя | негатив | код сигнала

— «полный отзыв / вопрос, где жирным выделен нужный кусок»

B. Все темы негативов

После полного списка сигналов сделай сводку:

тема — N/B — %

— имя

— «полная цитата с выделенным сигналом»

— имя

— «полная цитата с выделенным сигналом»

И так далее, сколько реально найдено.

C. Спорные случаи / ручная проверка

Показывай только если они есть:

имя | негатив | код сигнала

— причина неоднозначности

— найденные варианты отзывов целиком

D. Редкие, но важные негативы

Если среди негативных сигналов есть редкие, но особенно сильные минусы / риски / противопоказания / критические несоответствия, покажи их отдельно:

имя

— «полная цитата с выделенным сигналом»

E. Динамика по времени

Если в HTML-массиве у отзывов есть даты → 1–3 строки:

раньше vs сейчас

Если дат нет →

NO DATA

#### Финал

После полного результата выведи последней строкой только теги:

TAGS: тег (N/B, %); тег (N/B, %); тег (N/B, %)...

Не добавляй служебные и мета-теги
