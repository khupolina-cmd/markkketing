Ввод:

<<<

{СИГНАЛЫ}

Описание и обещания производителя:

<<<

{ОПИСАНИЕ_И_ОБЕЩАНИЯ_ПРОИЗВОДИТЕЛЯ}

HTML-массив отзывов / вопросов:

<<<

{HTML_МАССИВ}

Ниже я даю тебе уже зафиксированные сигналы в формате:

имя | категория | код сигнала

Пример:

Анна | позитив | ре.увл.

Твоя задача — не искать сигналы заново, а работать только с уже размеченными сигналами, которые вставлены в блок {СИГНАЛЫ}.

Что нужно сделать

Возьми только сигналы категории позитив.

Для каждого сигнала:

найди соответствующий отзыв / вопрос в HTML-массиве по имени;

разверни полный отзыв / вопрос целиком;

внутри полного текста жирным выдели только тот кусок, который соответствует сигналу;

если в одном отзыве несколько позитивных сигналов, разворачивай их по отдельности, а не одним блоком.

После полного разворота:

сгруппируй сигналы в темы по 2–4 слова;

посчитай частоты тем и отсортируй по убыванию;

посчитай B = общее число всех отзывов / вопросов в HTML-массиве;

для каждого финального тега посчитай % = (N / B) × 100.

Правила подсчёта

B = всё количество отзывов / вопросов в HTML-массиве, а не количество найденных сигналов;

N для темы = сколько разных отзывов / вопросов содержат эту конкретную тему;

если один и тот же сигнал повторяется в одном отзыве несколько раз, считай это как 1 отзыв;

если в одном отзыве есть несколько разных позитивных сигналов, каждый идёт в свой сигнал / тему отдельно;

если отзыв в целом негативный, но внутри есть позитивный сигнал, этот позитивный сигнал всё равно учитывается как позитивный;

проценты всегда считать от B, а не от числа сигналов и не от числа позитивных отзывов.

Правила работы

не переопределяй категорию сигнала;

не ищи новые сигналы;

не додумывай другие куски;

работай только с теми сигналами, которые вставлены в блок {СИГНАЛЫ};

если один и тот же отзыв разворачивается по нескольким позитивным сигналам, каждый раз показывай тот же полный текст, но с новым выделением;

не сокращай цитаты;

не пересказывай отзыв своими словами;

пустую похвалу типа «супер / класс / огонь / понравилось» без причины не считай ценностью, но учитывай в блоке возможной накрутки;

если имя в сигнале указано как «Покупатель», используй именно «Покупатель»;

если по имени в HTML найдено несколько отзывов и нельзя однозначно понять, какой относится к сигналу, не гадай и пометь как спорный случай в блоке ручной проверки.

Формат ответа

#### A. Все сырые позитивные сигналы

Для каждого сигнала:

имя | позитив | код сигнала

— «полный отзыв / вопрос, где жирным выделен нужный кусок»

Если в одном отзыве 3 позитивных сигнала, покажи этот отзыв 3 раза, каждый раз с новым выделением.

#### B. Все темы позитивных ценностей

После полного списка сигналов сделай сводку:

тема — N/B — %

— имя

— «полная цитата с выделенным сигналом»

— имя

— «полная цитата с выделенным сигналом»

И так далее, сколько реально найдено.

#### C. Обещания производителя → только подтверждения

Разбей блок {ОПИСАНИЕ_И_ОБЕЩАНИЯ_ПРОИЗВОДИТЕЛЯ} на список обещаний по 2–6 слов.

Для каждого обещания ищи среди уже размеченных позитивных сигналов только такие, которые подтверждают именно это обещание, а не смежную похвалу.

Строгое правило:

подтверждение засчитывай только если в отзыве есть тот же признак / факт, что и в обещании.

Если совпадение только общее — например: «классно», «понравилось», «приятный запах» — а обещание уже, точнее или про другой признак, не засчитывай.

Статус только 2 варианта:

подтверждается

подтвердить не удалось

Для каждого обещания:

обещание — подтверждается / подтвердить не удалось — N

Если подтверждения есть, дальше:

— имя

— «полная цитата с выделенным подтверждением»

— имя

— «полная цитата с выделенным подтверждением»

И так далее, сколько реально найдено.

Если блок производителя пустой, напиши:

обещания производителя не предоставлены

#### D. Сигнализация возможной накрутки хвалебных отзывов

Проверяй только то, что видно в тексте.

Даты и всплески не анализируй.

Проверь 4 вещи:

Оценочная однотипность — много пустой похвалы без смысла

Копипаст / шаблоны — одинаковые фразы, структура, эмодзи, пунктуация

Конкретика — много ли отзывов без деталей по сравнению с отзывами с деталями

Бонус / мотивация — баллы, подарок, кэшбек, промокод, скидка за отзыв, попросили оставить отзыв

Зона:

Зелёная — всё выглядит нормально → этот блок не писать

Жёлтая — есть странности → напиши:

нужно посмотреть вручную

Красная — сильные признаки накрутки → напиши:

высокий риск, смотрим вручную

Если зона жёлтая или красная — покажи, куда смотреть.

Копипаст:

3 крупнейшие группы похожих отзывов

что повторяется

сколько таких отзывов

все цитаты группы целиком

топ-5 повторяющихся фраз

Конкретика:

примерно доля отзывов без деталей

все цитаты без деталей целиком

все цитаты с деталями целиком

Бонус:

все найденные цитаты-сигналы целиком

#### E. Редкие, но важные позитивы

Если среди позитивных сигналов есть редкие, но особенно сильные ценности, покажи их отдельно:

имя

— «полная цитата с выделенным сигналом» — дата, если есть / [даты нет]

#### F. Динамика по времени

Если в HTML-массиве у отзывов есть даты → 1–3 строки:

раньше vs сейчас

Если дат нет →

NO DATA

Финал

После полного результата выведи последней строкой только теги:

TAGS: тег (N/B, %); тег (N/B, %); тег (N/B, %)...

Не добавляй служебные и мета-теги.
