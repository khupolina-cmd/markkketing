Ниже я даю тебе уже зафиксированные сигналы в формате:

Пример:

Мария | негатив | мят.уп.

Твоя задача — не искать сигналы заново, а работать только с уже размеченными сигналами, которые вставлены в блок {СИГНАЛЫ}.

#### Что нужно сделать

Для каждого сигнала:

найди соответствующий отзыв / вопрос в HTML-массиве по имени;

разверни полный отзыв / вопрос целиком;

внутри полного текста жирным выдели только тот кусок, который соответствует этому сигналу;

если в одном отзыве несколько сигналов, разворачивай их по отдельности, а не одним блоком.

После полного разворота:

сгруппируй сигналы не в короткие абстрактные темы, а в максимально конкретные почти-цитатные формулировки;

посчитай частоты этих формулировок и отсортируй по убыванию;

посчитай B = общее число всех отзывов / вопросов в HTML-массиве;

для каждой финальной формулировки посчитай % = (N / B) × 100.

#### Как делать формулировки

Финальная формулировка должна быть максимально близка к исходной реплике, почти как цитата.

Можно слегка подправить фразу для читаемости, но нельзя сжимать смысл и нельзя заменять конкретику абстракцией.

Главное правило:

не схлопывай смысл.

Формулировка должна быть такой, чтобы без чтения цитаты было сразу понятно, что именно человек имел в виду.

Плохо:

другой запах

для подарка

завышенные ожидания

аромат как условие

Хорошо:

ожидала другой запах

хотелось пожирнее и понажористей

хочу взять ещё на подарки, если аромат понравится

по описанию очень большие ожидания были

чтобы туба была не замята для подарка

#### Правила формулировок

формулировка может быть длинной;

формулировка должна быть конкретной и понятной;

формулировка должна быть максимально близка к исходной реплике;

не заменяй предмет сигнала обобщением;

не придумывай более короткую «аналитическую» версию;

не склеивай разные по смыслу сигналы только потому, что они из одной области;

объединяй сигналы только если у них реально почти одинаковое содержание.

#### Правила подсчёта

B = всё количество отзывов / вопросов в HTML-массиве, а не количество найденных сигналов;

N для формулировки = сколько разных отзывов / вопросов содержат эту конкретную формулировку;

если одна и та же формулировка повторяется в одном отзыве несколько раз, считай это как 1 отзыв;

если в одном отзыве есть несколько разных сигналов, каждый идёт в свою формулировку отдельно;

проценты всегда считать от B, а не от числа сигналов.

#### Правила работы

не переопределяй категорию сигнала;

не ищи новые сигналы;

не додумывай другие куски;

работай только с теми сигналами, которые вставлены в блок {СИГНАЛЫ};

если один и тот же отзыв разворачивается по нескольким сигналам, каждый раз показывай тот же полный текст, но с новым выделением;

не сокращай цитаты;

не пересказывай отзыв своими словами;

не заменяй почти-цитатные формулировки короткими тегами;

если имя в сигнале указано как «Покупатель», используй именно «Покупатель»;

если по имени в HTML найдено несколько отзывов и нельзя однозначно понять, какой относится к сигналу, не гадай и пометь как спорный случай в блоке ручной проверки.

#### 1) Что считать ожиданием

Ожидание = фрагмент отзыва или вопроса, где человек прямо формулирует:

что хотел / ожидал / рассчитывал / надеялся получить;

какой признак для него был важен: «важно было…», «хотелось…», «нужно было…», «для меня главное…»;

как товар должен был выглядеть / ощущаться / работать;

что он считал обещанным по карточке или описанию: «в карточке написано…», «заявлено…», «обещали…» — только если дальше есть, что именно человек ожидал получить.

Маркеры:

«ожидала»

«думала, что»

«надеялась»

«рассчитывала»

«хотела»

«хотелось»

«важно было»

«нужно было»

«должно было»

«по идее должно»

#### 2) Один отзыв может содержать несколько ожиданий

Не считай один отзыв за одно ожидание.

Один и тот же отзыв / вопрос может содержать несколько разных ожиданий сразу. Каждое ожидание нужно выделять отдельно и учитывать отдельно, даже если они находятся в одном и том же отзыве.

Пример:

если в одном отзыве есть:

ожидала более плотную текстуру

хотела не слишком сладкий аромат

рассчитывала на компактный формат для сумки

то это 3 разных сигнала ожидания, а не один.

#### 3) Как показывать сигнал

Для каждого найденного сигнала:

приводи полную цитату целиком;

жирным выделяй только тот кусок, который засчитан как ожидание;

один и тот же отзыв можно цитировать несколько раз, если в нём несколько ожиданий.

#### 4) Типы сигналов

Используй только эти типы:

[что хотел получить]

[важный критерий]

[как должно работать]

[как должно выглядеть / ощущаться]

[что считал обещанным]

#### 5) НЕ включай

Не включай:

сравнения / аналоги — это отдельный промпт;

сомнения / риски до покупки без ожидания — это промпт сомнений;

чистый постфактум-позитив / негатив без формулировки ожидания;

справочные вопросы без ожиданий;

путаницу / неясность карточки вида «это для рук или для тела?», если человек не формулирует, что именно он ожидал получить.

Такие фрагменты не считать ожиданием. Отправлять их в ручной разбор.
