Ввод:

<<<

{СИГНАЛЫ}

HTML-массив отзывов / вопросов:

<<<

{HTML_МАССИВ}

Ниже я даю тебе уже зафиксированные сигналы в формате:

имя | категория | код сигнала

Пример:

Ирина | сравнение | как.the.act

Твоя задача — не искать сравнения заново, а работать только с уже размеченными сигналами, которые вставлены в блок {СИГНАЛЫ}.

Что нужно сделать

Возьми только сигналы категории **сравнение**.

Для каждого сигнала:

- найди соответствующий отзыв / вопрос в HTML-массиве по имени;
- разверни полный отзыв / вопрос целиком;
- внутри полного текста **жирным** выдели только тот кусок, который соответствует сигналу сравнения;
- если в одном отзыве несколько сравнений, разворачивай их по отдельности, а не одним блоком;
- для каждого сигнала выдели точный **X**, с которым сравнивают продукт;
- если одна и та же сущность отличается только мелко — регистр, падеж, е/ё, незначительная орфографическая вариация — можно объединить в один **X**;
- посчитай, сколько раз каждый **X** встретился;
- посчитай **B = общее число всех отзывов / вопросов в HTML-массиве**;
- для каждого **X** посчитай **% = (N / B) × 100**.

Правила подсчёта

- **B** = всё количество отзывов / вопросов в HTML-массиве, а не количество найденных сравнений;
- **N для X** = сколько разных отзывов / вопросов содержат это конкретное сравнение;
- если один и тот же **X** повторяется в одном отзыве несколько раз, считай это как **1 отзыв**;
- если в одном отзыве есть 3 разных сравнения, каждый **X** учитывай отдельно;
- проценты всегда считать **от B**, а не от числа сравнений.

Правила работы

- не переопределяй категорию сигнала;
- не ищи новые сигналы;
- не додумывай другие куски;
- работай только с теми сигналами, которые вставлены в блок {СИГНАЛЫ};
- если один и тот же отзыв разворачивается по нескольким сравнениям, каждый раз показывай тот же полный текст, но с новым выделением;
- не сокращай цитаты;
- не пересказывай отзыв своими словами;
- не обобщай, не группируй и не переформулируй **X** сверх разрешённой нормализации;
- пиши только точные **X**, которые реально есть в тексте;
- если имя в сигнале указано как «Покупатель», используй именно «Покупатель»;
- если по имени в HTML найдено несколько отзывов и нельзя однозначно понять, какой относится к сигналу, не гадай и пометь как спорный случай в блоке ручной проверки.

Формат ответа

### A. Все сырые сигналы сравнений

Для каждого сигнала:

имя | сравнение | код сигнала

— X: ...

— «полный отзыв / вопрос, где **жирным** выделен нужный кусок»

Если в одном отзыве 3 сравнения, покажи этот отзыв 3 раза, каждый раз с новым выделением.

---

### B. Все X по убыванию частоты

Для каждого X:

X — N/B — %

— имя

— «полная цитата с **выделенным сравнением**»

— имя

— «полная цитата с **выделенным сравнением**»

И так далее, сколько реально найдено.

Расположи X по убыванию N.

---

### C. Ручной разбор

Если среди присланных сигналов есть блок **ручной разбор**, покажи его отдельно без перераспределения.

Формат:

имя | ручной разбор

— «полная цитата целиком»

Если ручного разбора нет — ничего не пиши.

---

### D. Критичные сигналы

Если среди сравнений есть критичные случаи:

- подмена;
- путаница вариантов;
- «вместо X пришло Y»;
- «не то»;

покажи их отдельно:

имя

— «полная цитата с **выделенным сравнением**» — дата, если есть / [даты нет]

---

### E. Динамика по времени

Если в HTML-массиве у отзывов есть даты → 1–3 строки:

`раньше vs сейчас`

Если дат нет →

`NO DATA`

---

Финал

После полного результата выведи последней строкой только теги:

`TAGS: X (N/B, %); X (N/B, %); X (N/B, %)...`

Не добавляй служебные и мета-теги.

Не обобщай **X** в категории. Пиши только точные сравнения из текста.
