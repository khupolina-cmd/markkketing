<<<

{СИГНАЛЫ}

HTML-массив отзывов / вопросов:

<<<

{HTML_МАССИВ}

Твоя задача — не искать всё подряд, а найти только сценарии покупки и использования: зачем человек купил товар, как собирается использовать, какую задачу решает, в каком контексте / для кого / когда.

Ничего не обрезай.

#### 1) Что считать сценарием

Сценарий = фрагмент отзыва или вопроса, где человек прямо или косвенно показывает:

зачем купил: «взяла для…», «искала для…», «нужен был…», «брала чтобы…»

как использует / планирует использовать: «ношу в сумке», «ставлю в ванной», «использую на ночь», «беру с собой»

для кого купил: «в подарок маме», «для ребёнка», «для мужа», «для гостей»

в каком контексте / сезоне / ситуации: «на зиму», «в офис», «в поездку», «для декора», «на подарок», «после мытья посуды», «когда сохнут руки»

какую проблему решает: «устала от старого аромата», «нужно было быстрое увлажнение», «хотелось что-то новое», «искала компактный формат»

Засчитывай и явные, и скрытые сценарии, если по фразе понятно:

почему человек выбрал товар;

где / когда / для кого он ему нужен;

какую практическую или эмоциональную задачу он закрывает.

#### 2) Один отзыв может содержать несколько сигналов

Не считай один отзыв за один сценарий.

Один и тот же отзыв / вопрос может содержать несколько разных сигналов сразу. Каждый сигнал нужно выделять отдельно и учитывать отдельно, даже если они находятся в одном и том же отзыве.

Пример:

если в одном отзыве есть:

купила в подарок маме

потом купила для себя

использую в сумочке

заказала ещё несколько ароматов

то это 4 разных сигнала, а не один.

#### 3) Как показывать сигнал

Для каждого найденного сигнала:

приводи полную цитату целиком;

жирным выделяй только тот кусок, который засчитан как сигнал;

один и тот же отзыв можно цитировать несколько раз, если в нём несколько сигналов.

#### 4) Типы сигналов

Используй только эти типы:

[зачем купил]

[как использует]

[для кого]

[контекст]

[решаемая задача]

#### 5) НЕ включай

Не включай:

просто позитив / негатив без сценария;

ожидания без указания применения или мотива покупки;

сравнения / аналоги без сценария;

сомнения / риски без сценария;

справочные вопросы без сценария.

#### 6) Анти-фантазия

Если фрагмент двусмысленный, спорный или требует догадки — не интерпретируй.

Перенеси в ручной разбор.

Формат:

имя | ручной разбор

— «полная цитата целиком» — причина: 3–7 слов

#### 7) Задача

Вытащи все сигналы сценариев из HTML-массива.

Один отзыв может дать несколько сигналов — учитывай каждый отдельно.

Для каждого сигнала покажи полную цитату и жирным выдели только нужный кусок.

Только после этого сгруппируй сигналы в темы по 2–4 слова.

Посчитай частоты тем и отсортируй по убыванию.

Посчитай B = общее число всех отзывов / вопросов в HTML-массиве.

Для каждого тега в финале посчитай % = (N / B) × 100.

#### 8) Правила подсчёта

B = всё количество отзывов / вопросов в HTML-массиве, а не количество найденных сигналов;

N для темы = сколько разных отзывов / вопросов содержат этот конкретный сценарий;

если один и тот же сигнал повторяется в одном отзыве несколько раз, считай это как 1 отзыв;

если в одном отзыве есть несколько разных сценарных сигналов, каждый учитывай отдельно;

проценты всегда считать от B, а не от числа сигналов.

#### 9) Формат ответа

#### A. Все сырые сигналы

Формат:

имя | [тип сигнала]

— «полная цитата с выделенным сигналом»

Если в одном отзыве 3 сигнала — покажи этот отзыв 3 раза, каждый раз с новым выделением.

#### B. Все темы сценариев

Формат:

тема — N/B — %

Дальше под темой:

— имя

— «полная цитата с выделенным сигналом»

— имя

— «полная цитата с выделенным сигналом»

И так далее, сколько реально найдено.

#### C. Ручной разбор

Формат:

имя | ручной разбор

— «полная цитата целиком» — причина: 3–7 слов

Если спорных случаев нет — ничего не пиши.

Если сценариев вообще не найдено, напиши:

в HTML-массиве не найдено

#### 10) Критичные сигналы

Критичные сигналы показывай всегда, если найдены:

не тот формат;

не тот товар;

не подходит под ключевой сценарий;

сломан главный сценарий использования;

высокий риск ошибки выбора из-за карточки товара.

Формат:

имя

— «полная цитата с выделенным сигналом» — дата, если есть / [даты нет]

#### 11) Динамика по времени

Если в HTML-массиве у отзывов есть даты → 1–3 строки:

раньше vs сейчас

Если дат нет →

NO DATA

#### 12) Финал

После полного результата выведи последней строкой только теги:

TAGS: тег (N/B, %); тег (N/B, %); тег (N/B, %)...

Не добавляй служебные и мета-теги.
